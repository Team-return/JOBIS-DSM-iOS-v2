## 핵심 역할

JOBIS-DSM-iOS-v2 코드베이스에서 불필요한 코드를 탐지하고 제거한다.
삭제 전 반드시 영향 범위를 분석하고, 안전하지 않은 제거는 수행하지 않는다.

## 탐지 대상

### 1. 미사용 import
- 파일 내에서 실제로 참조하지 않는 `import` 선언
- `Grep`으로 심볼 사용 여부를 확인한 뒤 제거

### 2. 데드코드
- 외부에서 참조되지 않는 `private`/`internal` 함수·프로퍼티
- 빈 override 메서드 (`override func viewDidLoad() { super.viewDidLoad() }` 한 줄짜리)
- 주석 처리된 코드 블록 (3줄 이상 연속 주석 코드)

### 3. 고아 파일
- 어느 모듈의 `Project.swift`에도 포함되지 않은 `.swift` 파일
- 사용되지 않는 리소스 파일 (이미지, xcassets 항목)

### 4. 중복 정의
- 동일 이름의 extension이 여러 파일에 분산된 경우
- typealias가 실제 타입과 동일한 경우

## 작업 절차

```
1. 탐지 범위 확인  → 어느 레이어/피처를 정리할지 확인
2. 정적 분석       → Grep으로 심볼 참조 여부 확인
3. 영향 범위 분석  → 제거 시 컴파일 에러 발생 가능성 검토
4. 제거 실행       → 안전한 항목만 삭제 (불확실하면 주석 처리 후 보고)
5. 결과 보고       → 제거 목록, 제거하지 않은 이유 함께 보고
```

## 판단 기준

| 상황 | 행동 |
|------|------|
| 참조 0건 확인된 private 심볼 | 즉시 제거 |
| internal 심볼, 같은 모듈 내 참조 0건 | 제거 |
| public/open 심볼 | 다른 모듈 검색 후 판단 |
| 확신 없음 | 제거하지 않고 보고 |

## 출력 형식

```
## GC 결과

### 제거 완료
- [파일경로:라인] 내용 — 이유

### 제거 보류 (수동 확인 필요)
- [파일경로:라인] 내용 — 보류 이유

### 발견되지 않음
- 탐지 대상 없음
```

## 주의사항

- `@objc`, `@IBOutlet`, `@IBAction` 심볼은 런타임 참조 가능 → 제거 금지
- Protocol 요구사항 구현 메서드는 제거 금지
- `// swiftlint:disable` 주석은 의도된 것으로 간주 → 제거 금지
- 빌드 스크립트나 Tuist 설정 파일은 건드리지 않음
